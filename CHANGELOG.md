## [0.1.8] - 2025-10-26

### 🐛 修复
- **挂载/卸载目录逻辑完全修复**：解决了DISM接收错误路径参数的根本问题
  - 修复WinPEBuilder中API调用错误，传递构建目录而非WIM文件路径
  - 修复LanguageConfig中挂载路径计算错误
  - 修复ISOCreator中卸载操作路径错误
  - 修复MountManager内部变量引用错误

### 🔧 改进
- **统一挂载目录规则**：所有操作遵循 `WIM文件.parent + "/mount"` 规则
  - WinPEBuilder: `current_build_path/media/sources/boot.wim` → `media/sources/mount`
  - WIM管理器: 扫描到的WIM文件路径 → WIM文件所在目录/mount
  - 构建管理器: boot.wim路径 → boot.wim所在目录/mount
- **卸载失败恢复机制增强**：
  - 实现多层次卸载失败恢复：重试 → 重新挂载 → 进程清理 → 强制删除
  - 针对错误代码0xc1420117（文件锁定）提供自动恢复方案
  - 详细的恢复过程日志记录

### 🚨 关键问题解决
- **路径传递错误彻底修复**：
  - 修复前：DISM接收构建目录路径 `D:\...\WinPE_20251026_200141`
  - 修复后：DISM接收正确WIM文件路径 `D:\...\WinPE_20251026_200141\media\sources\boot.wim`
  - 挂载目录统一为：`D:\...\WinPE_20251026_200141\media\sources\mount`

### 📚 技术改进
- **API调用统一化**：所有mount/unmount操作使用正确的`mount_winpe_image(wim_file_path)`方法
- **错误处理增强**：添加文件锁定检测、进程终止、强制删除等多种恢复策略
- **代码健壮性提升**：修复所有变量未定义、参数类型错误等底层问题

### 📊 修复范围
- **修复文件数**：6个核心文件
- **修复方法数**：8个关键方法
- **解决的问题**：挂载失败、卸载失败、路径错误、拒绝访问等关键问题
